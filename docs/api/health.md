## 服务健康检查

无需认证的公开端点，用于负载均衡器、监控系统探测服务可用性。

---

> 所有错误响应遵循统一格式，详见 [错误格式](./error.md)

### 获取服务健康状态

`GET /api/health`

**无需认证**。轻量级检查，仅确认进程在运行且可接受连接，不做数据库探测。

**响应** `200 OK`

```json
{
    "status": "ok",
    "timestamp": 1781700000000,
    "version": "0.1.0"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | 始终为 `"ok"`（服务可达） |
| `timestamp` | i64 | 服务器当前时间（13 位 Unix 毫秒时间戳） |
| `version` | string | 应用版本号 |

> 此端点与 `GET /api/admin/server/status` 不同：后者需要 `read_server` 权限并返回详细的系统/数据库/缓存状态信息。
